# Recorrer elementos con map
## Listar elementos sin usar map
- Si la lista de elementos es pequeña o estática, se pueden crear elementos <li> manualmente dentro de una lista <ul>:
```jsx
import imgUno from './assets/images/img-1.jpg'

const MyButton = () => {
  return (
    <button>Soy un boton</button>
  )
 }

const OnlineText = () =>{
  return (
    <div>
      <h3>Online</h3>
    </div>
  )
}
const OfflineText = () =>{
  return (
    <div>
      <h3>Offline</h3>
    </div>
  )
}
const App = () => {

  const title = 'Mi titulo desde una constante';
  const classTitle = 'text-center';
  // const pathImg = 'src/assets/images/img-1.jpg';
  const user = false;

  const fruts =['manzana', 'platano', 'pera'];

  return (
    <>
      <h1 className={classTitle}>{title}</h1>
      <img src={imgUno} alt={`imagen-${title}`} />
      <MyButton></MyButton>
      {user && <OnlineText/>}
      <ul>
        <li>{fruts[0]}</li>
        <li>{fruts[1]}</li>
        <li>{fruts[2]}</li>
      </ul>
    </>
  )
};

export default App;
```
### Ventajas:
- Sencillo y directo para pocas entradas.
### Limitaciones:
- No es práctico para listas dinámicas o con un número grande de elementos.

## Utilizar map para recorrer elementos
- Cuando la lista es dinámica o contiene muchos elementos, map es una herramienta eficaz para generar el contenido de forma más eficiente. Con map, puedes iterar sobre los elementos y devolver un componente o elemento HTML para cada uno:
```jsx
<ul>
  {fruts.map((frut) => (
    <li>{frut}</li>
  ))}
</ul>
```
### Consideraciones
- Cada elemento del array fruts genera un elemento <li> automáticamente.
- La sintaxis simplificada elimina las llaves {} y la palabra clave return cuando se devuelve un único elemento.
### Ventaja:
- Evita duplicación de código.
- Es flexible y escalable para listas dinámicas.
## Manejo de claves únicas (key)
- Al usar map en React, se genera una advertencia si no se proporciona una key única para cada elemento renderizado. Las claves permiten a React identificar de manera eficiente los elementos que cambian.
### Sin usar map:
- Adecuado para listas pequeñas y estáticas.
**![Normal](/React+vite-Fundamentos/imagenes/33.-recorrerMap.png)**

### Usando map con key:
- Soluciona las advertencias, mejora la eficiencia de React y es la práctica recomendada para listas dinámicas.
```jsx
import imgUno from './assets/images/img-1.jpg'

const MyButton = () => {
  return (
    <button>Soy un boton</button>
  )
 }

const OnlineText = () =>{
  return (
    <div>
      <h3>Online</h3>
    </div>
  )
}
const OfflineText = () =>{
  return (
    <div>
      <h3>Offline</h3>
    </div>
  )
}
const App = () => {

  const title = 'Mi titulo desde una constante';
  const classTitle = 'text-center';
  // const pathImg = 'src/assets/images/img-1.jpg';
  const user = false;

  const fruts =['manzana', 'platano', 'pera'];

  return (
    <>
      <h1 className={classTitle}>{title}</h1>
      <img src={imgUno} alt={`imagen-${title}`} />
      <MyButton></MyButton>
      {user && <OnlineText/>}
      <ul>
        {
          fruts.map((frut, index) => (
            <li key={index}>{frut} - {index}</li>
          ))
        }
      </ul>
    </>
  )
};

export default App;
```
