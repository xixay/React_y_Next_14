# 1. ¿Qué son los Hooks?
## 1.1. Definición
-Los Hooks son funciones que permiten a los desarrolladores utilizar características como el estado y el ciclo de vida de React en componentes funcionales. Antes de los Hooks, estas características solo estaban disponibles en componentes de clase.
## 1.2. Características principales de los Hooks
### 1.2.1. Compatibilidad con componentes funcionales:
- No necesitas clases para manejar el estado o el ciclo de vida. Todo se puede hacer dentro de funciones.
### 1.2.2. Facilidad de reutilización:
- Puedes crear tus propios Hooks para encapsular lógica reutilizable.
### 1.2.3. Hooks integrados:
- React ofrece varios Hooks como useState, useEffect, useContext, entre otros.
### 1.2.4. Inmutabilidad:
- Los Hooks están diseñados para trabajar con datos inmutables, lo que facilita la detección de cambios y asegura una renderización consistente.
## 1.3. Ejemplo: useState
### 1.3.1. Concepto de useState
- El Hook useState se utiliza para añadir estado local a un componente funcional. Cambiar este estado provoca que React vuelva a renderizar el componente con el nuevo valor.
### 1.3.2. Sintaxis Básica
```jsx
const [estado, setEstado] = useState(valorInicial);
```
- estado: Es la variable que contiene el valor actual del estado.
- setEstado: Es la función que actualiza el valor del estado.
- valorInicial: Es el valor con el que se inicializa el estado.
### 1.3.3. Ejemplo Práctico
- Código:
```jsx
import { useState } from 'react';

const Contador = () => {
  // Declaración del estado
  const [count, setCount] = useState(0);

  const incrementar = () => {
    setCount(count + 1); // Actualizar el estado
  };

  return (
    <div>
      <p>Has hecho clic {count} veces.</p>
      <button onClick={incrementar}>Incrementar</button>
    </div>
  );
};

export default Contador;
```
#### 1.3.3.1. Explicación
##### 1.3.3.1.1. Inicialización del estado:
```jsx
const [count, setCount] = useState(0);
```
- Se inicializa el estado count con un valor de 0.
##### 1.3.3.1.2. Actualización del estado:
```jsx
setCount(count + 1);
```
- La función setCount actualiza el valor de count. Esto notifica a React que debe re-renderizar el componente.
##### 1.3.3.1.3. Renderización
- React vuelve a renderizar el componente cada vez que cambia el estado, mostrando el valor actualizado de count.
#### 1.3.3.2. Resultado
- El botón muestra inicialmente: "Has hecho clic 0 veces".
- Cada clic en el botón incrementa el valor de count y actualiza la interfaz.
### 1.3.4. Ventajas de los Hooks
#### 1.3.4.1. Eliminación de clases:
- Puedes escribir componentes más simples y legibles.
#### 1.3.4.2. Reutilización de lógica:
- Los Hooks personalizados permiten encapsular lógica y compartirla entre componentes.
#### 1.3.4.3. Reducción de código repetitivo:
- Los Hooks ayudan a reducir el boilerplate necesario para manejar estado y ciclos de vida.
