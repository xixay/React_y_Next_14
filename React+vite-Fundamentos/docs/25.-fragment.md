# 1. Fragment
## 1.1. ¿Qué es un Fragment?
- Un Fragment permite agrupar múltiples elementos JSX sin añadir un nodo extra en el DOM, como un <div>.
## 1.2. Uso básico
- Puedes usar un Fragment vacío (<> </>) para evitar que aparezca un contenedor adicional:
```jsx
const App = () => {
  return (
    <>
      <h1 className="text-center">Hola soy react</h1>
      <img src="https://picsum.photos/200/300" alt="" />
    </>
  )
};

export default App;

```
- Resultado: El <div> que usualmente encierra los elementos desaparece.
**![Normal](/React+vite-Fundamentos/imagenes/25.-fragment.png)**

## 1.3. Llamar Fragment desde React
- Puedes importarlo como una exportación nombrada:
```jsx
import { Fragment } from "react";
const App = () => {
  return (
    <Fragment>
      <h1 className="text-center">Hola soy react</h1>
      <img src="https://picsum.photos/200/300" alt="" />
    </Fragment>
  )
};
export default App;
```
- O accederlo directamente como React.Fragment:
```jsx
import React from "react";
const App = () => {
  return (
    <React.Fragment>
      <h1 className="text-center">Hola soy react</h1>
      <img src="https://picsum.photos/200/300" alt="" />
    </React.Fragment>
  )
};
export default App;
```
## 1.4. Ventajas
- Ayuda a evitar crear nodos innecesarios en el DOM.
- Mantiene el código limpio y funcionalmente idéntico al uso de contenedores como <div>.
