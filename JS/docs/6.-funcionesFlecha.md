# funciones
## funcion normal
```js
function sumar(num) {
  console.log(10);
}
sumar(10); //10
```
## funcion flecha
```js
const sumarDos = (num1, num2) => {
  console.log(num1 + num2);
};
sumarDos(20, 30); //50
```
### Un solo parametro
- con solo un parametro se puede omitir parentesis
```js
const sumarTres = (num) => {
  console.log(num);
};
sumarTres(20); //20
```
### Sin parametros
- Los parentesis son obligatorios
```js
const saludar = () => console.log("Hola");
saludar(); //'Hola'
```
### Multiples parametros
- si se tiene return esta devolviendo un valor
```js
const sumarCuatro = (num1, num2) => {
  return num1 + num2;
};
const resultado = sumarCuatro(20, 30);
console.log(resultado); //50
```
### Un solo return
- si solo tiene un return se puede omitir las llaves
```js
const sumarCinco = (num1, num2) => num1 + num2;
const resultadoDos = sumarCinco(20, 30);
console.log(resultadoDos); //50
```
### Retornar un string
- retornar string
```js
const mensaje = () => {
  return 'hola soy xixay';
};
const resultadoTres = mensaje();
console.log(resultadoTres); //'hola soy xixay'
```
#### Un solo parametro
- si se tiene solo un parametro se puede omitir los parentesis
```js
const mensajeDos = nombre => {
  return 'hola soy ' + nombre;
};
const resultadoCuatro = mensajeDos('Yumiko');
console.log(resultadoCuatro); //'hola soy Yumiko'
```
#### Un solo return
- si se tiene un solo return se puede omitir los parentesis y el return
```js
const mensajeTres = nombre => 'hola soy ' + nombre;
const resultadoCinco = mensajeTres('sunset');
console.log(resultadoCinco); //'hola soy sunset'
```
### otros ejemplos
#### Sin paréntesis (simplificada)
- Esta es la forma más concisa y común cuando la función tiene un solo retorno y no necesita estructuras complejas:
```js
const mensaje = (nombre) => 'Hola soy ' + nombre;

const resultado = mensaje('Shimmer');
console.log(resultado); // 'Hola soy Shimmer'
```
#### Con paréntesis (opcional para mejorar legibilidad o manejar estructuras complejas):
- Los paréntesis son útiles para claridad visual, especialmente cuando se trabaja con múltiples líneas o estructuras complejas como objetos o JSX:
```js
const mensaje = (nombre) => (
  'Hola soy ' + nombre
);

const resultado = mensaje('Shimmer');
console.log(resultado); // 'Hola soy Shimmer'
```
#### Uso obligatorio de parentesis
- Cuando retornas un objeto literal:
```js
const crearObjeto = () => ({
  clave: 'valor',
  numero: 42,
});
console.log(crearObjeto()); // { clave: 'valor', numero: 42 }
```
- Cuando retornas JSX en React:
```js
const MiComponente = () => (
  <div>
    <h1>Hola, soy un componente</h1>
  </div>
);
```
### Valores por defecto
- valores por defecto para que no este undefined
```js
//por defecto num sera uno
const sumaSeis = (num = 1) => {
  console.log(num + 3); //4
};
sumaSeis();
```
